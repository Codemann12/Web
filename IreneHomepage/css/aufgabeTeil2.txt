5. Separation of Concerns ist in Java nicht umgesetzt. Das Prinzip wird oft in Projektmanagement benutzt, um effektive Teams zu bilden.


6. Änderbarkeit ist in Java unterstützt. Jeder Java Code kann geändert werden und nur so ist die Wartung von Software, die in Java geschrieben sind möglich.


7. Inkrementalität ist auch in Java unterstützt. Einzelne Software-Komponenten können in einem Packet (package) implementiert werden, bevor alle Komponenten entwickelt werden.


8. Gründlichkeit und Formalität ist nur teilweise unterstützt. Nur die formalen Dokumentation wird hier durch javadoc unterstützt und den Rest(Formaler Entwurf,...) nicht.
